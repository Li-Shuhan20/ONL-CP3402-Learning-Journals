# Content Insights

The client sends requests to the server, and the server processes these requests and returns resources. A CMS like WordPress resides on the server side, dynamically generating these resources from content stored in a database rather than serving static HTML files.

Browser each have their own rendering engines, which interpret HTML, CSS, and JavaScript differently. This explains why cross-browser compatibility is such an important consideration when designing websites — a concern that CMS themes also need to handle properly.
