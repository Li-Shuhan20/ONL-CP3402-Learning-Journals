## Content Insights
- The relationship between the parent and child themes means that the child theme inherits all the functions and styles, but can override specific elements. This forms a secure modification layer - updates to the parent theme are passed down, but the custom content in the child theme remains unchanged.
- Using development tools for browser-based development has good efficiency. Modifying CSS directly in the inspector can provide immediate visual feedback across different screen sizes, and it can also use the responsive design mode. Once the styles are optimized, copying them back into the CSS file of the child theme ensures that these changes become permanent.

## Career/Employability/Learning Insights
- Understanding sub-themes is like learning the best practices of the industry, rather than merely a function. It is possible to safely expand existing themes without disrupting future updates.
- The browser development workflow guide is very practical and has provided me with some assistance. Previously, when making CSS changes in the code editor, I would save the changes, switch to the browser, and refresh - often repeating this cycle. However, using the development tools to conduct experiments first, and then copying the effective code back, can save a significant amount of time. This improvement in efficiency means that I can iterate faster and try more design changes within the same amount of time.
